## Executive Summary

### 왜 바꾸는가

현재 모델(Random Forest, R² 90.1%)은 차량 스펙을 넣으면 낙찰가를 한 번에 예측하는 **블랙박스** 방식이다. 실무 피드백은 "예측이 맞지 않는다." 그 이유:

| 문제 | 영향 |
|------|------|
| 차명/트림 정보 미사용 | "그랜저 프리미엄"과 "그랜저 캘리그래피"를 구분 못 함 |
| 수출 51% 섞임 | 내수가와 수출가 가격 구조가 완전히 다름 (고키로 시 2배 차이) |
| 렌터카 48% 미구분 | 차종별 -25%~+25% 편차 → 단일 모델로 불가 |
| 사고이력 미반영 | 교환/판금 3개 이상 시 -5~8% 급감 |
| 블랙박스 | 왜 그 가격인지 설명 불가 → 전문가가 신뢰하지 않음 |

### 어떻게 바꾸는가

프라이싱 매니저의 **실제 감정 프로세스**를 시스템화:

```
[현재: v1]                          [개선: v2]
차량 스펙 입력                      대상차량 입력
     ↓                                  ↓
ML 모델 (블랙박스)                  LLM 리즈닝 모델 (도메인 지식 + DB 검색)
     ↓                                  ↓
예측가 출력                         기준차량 3건 추천 + 선택 이유
                                         ↓
                                    프라이싱 매니저 확인/수정
                                         ↓
                                    룰 엔진 보정 (단계별 표시)
                                         ↓
                                    최종 가격 산출 (소매가/예상 낙찰가)
```

### 핵심 차이

| 항목 | v1 (현재) | v2 (개선) |
|------|-----------|-----------|
| **모델** | Random Forest 회귀 | LLM 리즈닝 + 룰 엔진 |
| **기준차량** | 없음 (직접 예측) | 유사 낙찰 차량 3건 추천 |
| **보정 과정** | 블랙박스 | 단계별 투명하게 표시 |
| **설명 가능성** | 없음 | LLM이 자연어로 이유 설명 |
| **전문가 역할** | 결과만 확인 | 기준차량 선택에 참여 |
| **데이터 활용** | 8개 기본 특성 | 차명/트림/사고이력/경력/수출구분 전부 활용 |
| **프론트엔드** | Streamlit | React + FastAPI |
| **자동화** | 완전 자동 (부정확) | Human-in-the-loop → 점진적 자동화 |

### 6. LLM 기준차량 추천 서비스 ✅

프라이싱 매니저의 실제 추론 프로세스를 LLM으로 구현.

**핵심 설계:**
- 기준차 선택 우선순위: 모델+트림(필수) → 연식(매우 중요) → 옵션(교차검증) → 주행거리(보정 가능)
- **삼각측량**: 옵션 구성이 조금씩 다른 3건을 추천하여 옵션 가치를 교차검증
- LLM이 3개 도구(`search_auction_db`, `get_price_stats`, `get_vehicle_detail`)를 자유롭게 호출하며 추론
- Agentic loop: 최대 10회 도구 호출 → 최종 추천 3건 + 이유 설명

**산출물:**
- `backend/app/services/llm_recommender.py` — LLM 추천 서비스 (시스템 프롬프트 + 도구 + agentic loop)
- `backend/app/api/recommend.py` — 추천 API (`POST /api/recommend`)

## Phase 2: 기준차량 추천 — LLM 리즈닝 모델 (2~3주)

### 핵심 아이디어

프라이싱 매니저가 머릿속에서 하는 추론을 **LLM 리즈닝 모델**로 구현.
모델이 경매 데이터, 도메인 지식, 실제 사례를 종합 추론하여 기준차량 3건을 추천 + 선택 이유를 설명.

```
[대상차량 입력]
     ↓
[DB 검색 도구] → 후보 20~30건 pre-filter
     ↓
[LLM 리즈닝 모델]
  ├── 프라이싱 룰 & 노하우 (시스템 프롬프트)
  ├── 한국 차량 시장 지식
  ├── 기준차 선택 퓨샷 예시 (전문가 사례)
  ├── 차량 분류 체계 (taxonomy)
  │
  ├── [도구1] 경매 DB 검색
  ├── [도구2] 시세 통계 조회
  └── [도구3] (향후) 유사 사례 벡터 검색
     ↓
[Top 3 추천 + 선택 이유]
     ↓
[프라이싱 매니저 확인/수정] → 선택 이력 저장
     ↓
[룰 엔진 가격 산출]
```

### LLM에 주입하는 도메인 지식

| 영역 | 내용 | 예시 |
|------|------|------|
| **프라이싱 룰** | 검증된 보정 룰 전체 | 주행거리 감가, 색상 보정, 교환/판금 등 |
| **한국 시장 지식** | 브랜드/세그먼트/소비자 선호 | 르쌍셰 비선호, 대형차 흰색 프리미엄 |
| **퓨샷 예시** | 전문가의 실제 기준차 선택 사례 | "GN7 프리미엄 → 이런 기준차를 골랐다" |
| **차량 분류 체계** | taxonomy + 트림 계층 | 기본 < 프리미엄 < 익스클루시브 < 캘리그래피 |

### LLM이 사용하는 도구

```python
# 도구 1: 낙찰 DB 검색
search_auction_db(maker, model, generation, trim, year_range, mileage, ...)
→ 조건에 맞는 낙찰 차량 목록 반환

# 도구 2: 시세 통계
get_price_stats(maker, model, generation, year, months=3)
→ 최근 N개월 평균/중앙/최고/최저 낙찰가

# 도구 3 (향후): 유사 사례 벡터 검색
search_similar_cases(target_vehicle, top_k=5)
→ 과거 전문가 선택 이력에서 유사 패턴 추출
```

### LLM 추론 흐름 예시

1. "그랜저 GN7 2023 프리미엄 3만km 자가용이구나"
2. `search_auction_db(maker="현대", model="그랜저", generation="GN7", trim="프리미엄")` 호출
3. 결과 20건 → "동일 트림 8건, 비슷한 주행거리 3건"
4. `get_price_stats()`로 시세 맥락 확인
5. 도메인 지식 + 퓨샷 참조 → Top 3 선정 + 이유 설명

### Human-in-the-Loop

- LLM이 **Top 3 추천 + 각각의 선택 이유** 자연어로 제공
- 프라이싱 매니저가 선택하거나 "더 보기"로 확장
- 선택 이력 저장 → 퓨샷 풀 확장 + 향후 벡터 서치 학습 데이터

### 향후 확장: 벡터 서치

선택 이력이 100건+ 쌓이면 LLM에 추가 도구로 제공:
- "비슷한 차량을 감정할 때 전문가들은 이런 기준차를 골랐다"
- 리즈닝 모델이 필요 시 자발적으로 호출 (강제 아님)

## Phase 3: 룰 엔진 (2~3주)

### 확보된 보정 룰

```
[기준차량 → 대상차량 보정]

1. 교환(X): 비선형 — 1~2개 무시, 3개+ 5%, 5개+ 10%  ← 데이터 검증 반영
2. 판금·도색(W): 연식/가격대별 1~2% (비선형 적용)
3. 연식: 기준차량 대비 연식별 가격차
4. 주행거리: 연식별 차등 적용 (데이터 검증된 수치)
   - 2~3년: 2.7%/만km (차종별 2.4~3.2%)
   - 4~6년: 1.9%/만km
   - 7~9년: 2.0%/만km (SUV 1.2%, 세단 3.0%)
   - 10년+: 14만원/만km (K5 18만원)
   - 20만km+: 둔화 적용 (완전 정체 아님)
5. 색상: 차급별 차등 (대형 최대 268만원, 경차 28만원)
6. 렌터카: 차종별 개별 계수 (-25% K5 ~ +25% 스타렉스)
7. 수출/내수: 기본 내수만, 20만km+ 수출 별도 처리
8. 옵션(선스네후): 차급별 차등 (중형 41만 / 대형 154만 / SUV 182만)
9. 가격 산출: 소매가 × 90% (외제차 88%), 최소 마진 150만원
```

### 룰 데이터 검증 — ✅ 완료

9개 항목 검증 완료 (상세 결과: 부록 참조). 대부분의 업계 룰이 수정 필요하며, 검증된 실제 파라미터를 적용.

### 추가 확보 필요한 룰

- 차종별 특수 룰 ("이 차종은 옵션 안 탄다" 등)
- 브랜드 비선호 계수 세분화 (르노, 쌍용, 쉐보레)
- 계절성 / 시장 트렌드 반영
- 연식 경계 효과 (12월→1월 전환)

### 룰 저장 구조

- JSON/YAML로 구조화 → 비개발자(프라이싱 매니저)도 수정 가능
- 버전 관리 (변경 이력 추적)
- 차종별 오버라이드 (일반 룰 + 차종 특수 룰)

## Phase 4: 프론트엔드 — React + FastAPI (Phase 2~3과 병행)

### 시스템 아키텍처

```
[React Frontend]
     ↕
[FastAPI Backend]
     ├── POST /api/recommend    → LLM 리즈닝 모델 호출
     │                             ├── Tool: DB 검색
     │                             ├── Tool: 시세 통계
     │                             └── Tool: (향후) 벡터 서치
     ├── POST /api/calculate    → 룰 엔진 가격 산출
     ├── POST /api/feedback     → 선택 이력 + 피드백 저장
     └── GET  /api/vehicle-info → 차명 자동완성 (taxonomy)
     ↕
[DB]
     ├── 낙찰 DB (58,252건+)
     ├── 신차가격 DB
     ├── 선택 이력 DB
     └── 룰 파라미터 (JSON/YAML)
```

### 주요 화면

| 화면 | 설명 |
|------|------|
| **대상차량 입력** | 차종/트림 자동완성 (taxonomy), 연식, 주행거리, 사고, 옵션 |
| **기준차량 추천** | LLM Top 3 카드 + 선택 이유 자연어 설명 |
| **보정 과정** | 기준차 → 각 룰 적용 단계별 투명 표시 |
| **최종 결과** | 소매가 / 예상 낙찰가 + 신뢰도 + 보정 내역 |
| **피드백** | 기준차 적합성 + 가격 적절성 + 실제 낙찰가 |

### 검증

## 부록: 데이터 검증 상세 결과

### A. 주행거리 감가율

| 연식 | 업계 룰 | 실제 평균 | 차종별 범위 | 판정 |
|------|---------|----------|-----------|------|
| 2~3년 | 2%/만km | **2.7%** | 2.4~3.2% | 과소 |
| 4~6년 | 1.5%/만km | **1.9%** | 1.5~2.4% | 약간 과소 |
| 7~9년 | 1%/만km | **2.0%** | 1.2~3.0% | 과소 |
| 10년+ | 1% or 10만원 | **2.1% / 14만원** | 10~18만원 | 과소 |

- SUV(싼타페/쏘렌토)가 세단보다 감가에 강함 — 7~9년: 싼타페 1.2% vs K5 3.0%
- 10만km 전후에서 급격한 감가 → 비선형 커브 필요

### B. 20만km 천장 효과

- 20~30만km 구간에서 감가 기울기 현저히 둔화 (구간당 5~9%)
- 완전한 정체(flat)는 아님 — 20만→30만에도 꾸준히 하락
- 30만km+ 표본 부족 (18건)으로 통계적 신뢰도 낮음

### C. 색상 효과

| 차급 | 데이터 결과 | 업계 룰과 비교 |
|------|-----------|--------------|
| **대형** | 흰색 +70만 >> 메탈 > 검정 >> 은색 -200만 | 격차 268만원 (20만원 단위 아님) |
| **중형** | 메탈 +38만 > 검정 > 은색 > 흰색 | 흰색이 최하위 — 룰과 정반대 |
| **경차** | 검정 +19만 > 메탈 > 은색 > 흰색 | 흰색 1위 룰과 불일치 |

- G80에서 흰색 프리미엄 +169만원 (극단적)
- 색상 효과는 차급이 올라갈수록 격차 확대 (경차 28만 vs 대형 268만)

### D. 브랜드 비선호

- 르노/쌍용/쉐보레: 시작가 자체가 이미 낮게 책정
- 절대 잔존가치: 동급 현대/기아 대비 600~1,000만원 낮음
- 감가 속도(비율)보다 출발점(신차가 대비 잔존가)의 차이가 핵심

### E. 선호옵션(선스네후)

| 옵션 | 중형(K5) | 대형(그랜져) | SUV(싼타페) |
|------|---------|-----------|-----------|
| 선루프 (트림 보정 후) | +41만 | +154만 | +182만 |
| 4개 풀장착 vs 0개 | +304만 | +336만 | +307만 |

### F. 교환/판금

| 항목 | 업계 룰 | 실제 | 판정 |
|------|---------|------|------|
| 교환 1회당 | 3% | 1.7% | 과대 |
| 판금 1회당 | 1~2% | 1.9% | 부합 |
| 긁힘 | - | 영향 없음 | 무시 |

- 비선형: 1~2개 거의 영향 없음, 3개 이상에서 -5~8% 급감

### G. 수출/내수

| 주행거리 | 가격 관계 | 차이 |
|----------|----------|------|
| ~15만km | 내수 > 수출 | 내수 10~20% 높음 |
| 15~20만km | 역전 시작 | 수출 4% 높음 |
| 20만km+ | 수출 >> 내수 | 수출 33% 높음 |

### H. 렌터카

| 차종 | 렌터카 할인율 (주행거리 보정 후) |
|------|-------------------------------|
| K5 | -25.1% |
| 쏘나타 | -17.4% |
| 그랜져 | -12.7% |
| 싼타페 | -5.8% |
| 스타렉스 | +25.4% (역전) |
