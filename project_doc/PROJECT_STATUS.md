# 자동차 경매 가격 예측 ML 프로젝트

**최종 업데이트**: 2026년 2월 8일

## 1. 프로젝트 개요

### 1.1 목표
숙련된 자동차 감정사가 2-3년의 경험을 통해 습득하는 **경매 낙찰가 예측 능력**을 머신러닝으로 자동화

### 1.2 핵심 질문
- 차량 정보(연식, 주행거리, 옵션 등)만으로 낙찰가를 얼마나 정확히 예측할 수 있는가?
- 어떤 가격대의 차량이 예측하기 쉬운가/어려운가?
- 이 예측을 활용한 실전 구매 전략은 무엇인가?

### 1.3 현재 상태
| 항목 | 상태 | 비고 |
|------|------|------|
| 데이터 수집 | ✅ 완료 | 58,252건 (중복 제거 후) |
| 데이터 정제 | ✅ 완료 | 유니코드 정규화, 중복 제거 |
| 모델 학습 | ✅ 완료 | Random Forest (R² 90.1%) |
| 성능 분석 | ✅ 완료 | 가격대별 MAPE, 기대값 분석 |
| 보고서 작성 | ✅ 완료 | v5 (데이터 정제 반영) |
| 웹앱 프로토타입 | ✅ 완료 | Streamlit 기반 (app.py) |
| 웹 배포 | ✅ 완료 | Streamlit Cloud |
| 사용자 테스트 | 🔜 진행중 | 감정사 피드백 수집 |

## 2. 데이터

### 2.1 데이터 소스
```
경매데이터/
├── 20250525~20250611 낙찰데이터.xlsx     # 메인 파일 (47,642건)
├── 경매 전체 데이터 (성능포함) 25.xx.xx.xls  # 주간 데이터 (10개 파일)
└── (기타 중복/피벗 파일들 - 제외됨)
```

### 2.2 데이터 현황
| 항목 | 수치 |
|------|------|
| 총 로드 데이터 | 67,762건 |
| 학습 대상 (낙찰 차량) | 58,252건 |
| 학습 세트 | 46,601건 (80%) |
| 테스트 세트 | 11,651건 (20%) |
| 데이터 기간 | 2024-07-03 ~ 2025-06-11 (약 11개월) |
| 평균 낙찰가 | 1,295만원 |

### 2.3 데이터 정제 이슈 (해결됨)

#### 이슈 1: 중복 파일 로딩
- **문제**: `낙찰데이터.xlsx`와 `낙찰데이터 1-4.xlsx`가 동일 데이터의 분할본
- **영향**: 115,404건 로드 → 실제 58,252건 (약 48,000건 중복)
- **결과**: R² 94.7% → 90.1% (과대평가 수정)

#### 이슈 2: macOS 유니코드 (NFD/NFC)
- **문제**: macOS가 한글 파일명을 NFD(분해형)로 저장
- **영향**: Python의 문자열 매칭 실패
- **해결**: `unicodedata.normalize('NFC', filename)` 적용

### 2.4 사용된 특성 (24개)

**기본 특성 (8개)**
- 연식, 주행거리, 제작사, 차종, 변속기, 연료, 색상, 종목

**파생 특성 (2개)**
- 차량나이 (2025 - 연식)
- 연평균주행 (주행거리 / 차량나이)

**옵션 특성 (14개)**
- 네비게이션, 후방카메라, 열선시트, 통풍시트, 가죽시트
- 선루프, 스마트키, HUD, 어라운드뷰, LED헤드램프
- 크루즈컨트롤, 자동주차, 차선이탈, 파워트렁크

## 3. 모델 성능

### 3.1 모델 비교 (테스트 세트 기준)

| 순위 | 모델 | R² | RMSE | MAE | MAPE | 학습시간 |
|------|------|-----|------|-----|------|----------|
| 🥇 | **Random Forest** | **90.1%** | 334만원 | 163만원 | 16.2% | 2.6초 |
| 🥈 | XGBoost | 90.0% | 336만원 | 182만원 | 18.7% | 0.1초 |
| 🥉 | LightGBM | 89.7% | 341만원 | 187만원 | 19.4% | 0.1초 |
| 4 | CatBoost | 88.1% | 367만원 | 199만원 | 20.7% | 0.5초 |

**채택 모델**: Random Forest (R² 90.1%)

### 3.2 특성 중요도 (CatBoost 기준)

| 순위 | 특성 | 중요도 |
|------|------|--------|
| 1 | 제작사 | 20.8% |
| 2 | 연식 | 13.6% |
| 3 | 차종 | 13.5% |
| 4 | **옵션_파워트렁크** | 11.5% |
| 5 | 차량나이 | 11.1% |
| 6 | 주행거리 | 9.7% |
| 7 | 옵션_HUD | 5.0% |
| 8 | 옵션_어라운드뷰 | 3.8% |

**인사이트**: 옵션 특성이 전체 중요도의 약 27%를 차지하며, 특히 파워트렁크(11.5%)가 가장 중요한 옵션

## 6. 사용 방법

### 6.1 모델 학습
```bash
cd car_price_prediction
pip install -r requirements.txt
python train_model.py
```

### 6.2 가격 예측 (CLI)
```bash
python predict.py
# 대화형으로 차량 정보 입력 → 예측가 출력
```

### 6.3 웹앱 실행
```bash
streamlit run app.py
# 브라우저에서 http://localhost:8501 접속
```

## 7. 웹앱 상세 (app.py)

### 7.1 기능 목록

| 기능 | 설명 |
|------|------|
| **입력 폼** | 제작사, 차종, 연식, 주행거리, 연료, 변속기, 색상, 종목 |
| **옵션 선택** | 14개 옵션 체크박스 (파워트렁크, HUD, 어라운드뷰 등) |
| **예측 결과** | 예상 낙찰가 + MAPE(신뢰도) 표시 |
| **구매 가이드** | 75%/60%/90% 수익 확률별 최대 입찰가 |
| **피드백 수집** | 👍적절 / 📈낮음 / 📉높음 버튼 |
| **상세 피드백** | 실제 예상가 입력 + 코멘트 |
| **사이드바** | 모델 정보, 가격대별 정확도 표시 |

### 7.2 피드백 저장 형식

```json
{
  "timestamp": "2025-02-08T12:34:56",
  "inputs": {
    "제작사": "현대",
    "차종": "SUV",
    "연식": 2020,
    "주행거리": 50000,
    "옵션_list": ["네비게이션", "후방카메라"]
  },
  "predicted_price": 1500.0,
  "feedback_type": "적절",
  "actual_price": null,
  "comment": null
}
```

저장 위치: `output/feedback.jsonl`
